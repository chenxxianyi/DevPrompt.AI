# 当前仓库修复指引

更新时间：2026-05-10

这份文档记录的是当前仓库最值得优先处理的工程问题，目的是让后续 PR 更聚焦、更可验收。

## P0

### 1. 乱码文案与历史编码问题

范围：

- 前端页面文本
- Go 日志与种子文案
- 旧版说明文档

影响：

- 阅读成本高
- 容易误判真实功能

建议：

- 新改动统一使用 UTF-8
- 逐模块清理用户可见文本
- 修复时以当前功能不变为前提

### 2. `database/init.sql` 与运行时种子逻辑漂移

现状：

- SQL 初始化脚本仍然存在
- 服务启动又会执行 `AutoMigrate + Seed`

影响：

- 维护两套初始化事实来源
- 容易让环境表现不一致

建议：

- 明确一套权威初始化路径
- 若继续保留 SQL，需定期与模型和种子逻辑对齐
