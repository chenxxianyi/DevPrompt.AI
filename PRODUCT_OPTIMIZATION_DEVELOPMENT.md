# DevPrompt AI 产品优化开发说明

更新时间：2026-05-10

## 产品定位

DevPrompt AI 当前不是单一的 Prompt 输入框，而是一套面向开发者的 Prompt 工作台：

- 用模板发现可复用经验
- 用生成器产出结构化 Prompt
- 用历史记录和个人工作台沉淀复用资产
- 用后台管理内容、模型和会员配置

## 当前产品面

### 用户端

- 首页快速生成
- 模板列表与详情
- 项目 Prompt 生成
- Cursor Rules 生成
- Claude Code Prompt 生成
- Prompt 优化
- 生成历史
- 个人工作台
- 会员方案与试用申请

### 管理后台

- 用户管理
- 分类管理
- Prompt 管理
- AI 模型管理
- AI 调用日志
- 会员方案管理
- 订单查看
- 项目类型管理

### 服务端

- JWT 登录注册
- Prompt 模板与交互
- 四类生成接口
- 用户额度统计
- 试用申请接口
- 管理接口

## 当前产品缺口

### 1. 试用申请闭环还差最后一段

现状：

- 用户端已能提交试用申请
- 服务端已能处理试用申请
- 管理后台还没有对应页面

影响：

- 线索进入系统后无法在前端后台闭环处理

### 2. 模板资产化还不够深

现状：

- 模板有基础分类、详情、点赞、收藏
- 但缺少更强的“适用工具”和“使用示例”表达

影响：

- 模板价值更多停留在浏览层面，复用效率一般

### 3. Admin 仍偏配置台

现状：

- 后台已覆盖大部分基础 CRUD
- 仪表盘仍然偏轻量

影响：

- 还不足以支持运营分析和转化跟踪

### 4. 部署心智不够统一

现状：

- 用户端已进入 Compose
- 后台需要单独本地运行

影响：

- 新成员容易误以为 `/admin` 已有完整部署

## 开发建议

### 第一阶段：补齐闭环

- 为后台增加试用申请管理页面
- 会员页补联系人和备注输入
- 在个人工作台增加更明显的会员升级引导

### 第二阶段：提升模板复用效率

- 增加模板适用工具标签
- 增加模板示例输入与示例输出
- 支持从模板详情更顺滑地进入对应生成场景

### 第三阶段：增强运营能力

- 扩展后台仪表盘
- 增加试用申请统计
- 关联订单、会员和生成行为

### 第四阶段：规范部署

- 为 `admin` 增加标准部署方案
- 明确正式环境的访问路径和反向代理结构

## 版本建议

| 版本 | 目标 |
| --- | --- |
| `v0.5.x` | 补齐试用申请闭环 |
| `v0.6.x` | 提升模板资产化与后台运营能力 |
| `v0.7.x` | 完成 Admin 标准化部署与更多指标建设 |

## 验收重点

- 用户从会员页提交申请后，运营能在后台跟进
- 新用户按文档可在 30 分钟内完成本地跑通
- 模板页面能更清楚表达“适用于什么工具、适用于什么场景”
